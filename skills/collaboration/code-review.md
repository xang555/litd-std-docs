# Code Review

## Description

Best practices for conducting and participating in code reviews.

## Level: Junior

### Learning Objectives

- Understand the purpose of code review
- Learn to review code effectively
- Receive feedback constructively

### Required Skills

1. **Review Basics**

   - Why we review code
   - Review checklist
   - Common issues to look for

2. **Reviewing Code**

   - Understanding context
   - Checking logic correctness
   - Identifying style issues

3. **Receiving Feedback**

   - Open mindset
   - Asking questions
   - Implementing changes

### Resources

- Code review guidelines
- Team review checklist

## Level: Mid-Level

### Learning Objectives

- Provide actionable feedback
- Review for architecture
- Mentor through review

### Required Skills

1. **Effective Feedback**

   - Clear, specific comments
   - Explain reasoning
   - Suggest improvements

2. **Architectural Review**

   - Design patterns
   - Coupling and cohesion
   - Scalability concerns

3. **Mentorship**

   - Teaching moments
   - Knowledge sharing
   - Constructive criticism

### Optional Skills

- Security-focused review
- Performance review
- Accessibility review

## Level: Senior

### Learning Objectives

- Lead review culture
- Improve review processes
- Balance standards with pragmatism

### Required Skills

1. **Review Leadership**

   - Setting standards
   - Encouraging participation
   - Handling disagreements

2. **Process Improvement**

   - Review metrics
   - Tool optimization
   - Workflow refinement

3. **Team Growth**

   - Training reviewers
   - Building habits
   - Quality culture

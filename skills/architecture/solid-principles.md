# SOLID Principles

## Description

The five principles of object-oriented design for maintainable software.

## Level: Junior

### Learning Objectives

- Understand the five SOLID principles
- Recognize violations
- Apply principles to simple code

### Required Skills

1. **Single Responsibility**

   - One reason to change
   - Focused classes/functions
   - Cohesion

2. **Open/Closed**

   - Open for extension
   - Closed for modification
   - Abstraction usage

3. **Liskov Substitution**

   - Subtype behavior
   - Contract adherence
   - Inheritance design

4. **Interface Segregation**

   - Focused interfaces
   - No fat interfaces
   - Client-specific contracts

5. **Dependency Inversion**

   - Depend on abstractions
   - Injection patterns
   - Loose coupling

# Design Patterns

## Description

Common design patterns for solving recurring software problems.

## Level: Junior

### Learning Objectives

- Understand common design patterns
- Recognize when to apply patterns
- Implement basic patterns

### Required Skills

1. **Creational Patterns**

   - Singleton
   - Factory
   - Builder

2. **Structural Patterns**

   - Adapter
   - Decorator
   - Facade

3. **Behavioral Patterns**

   - Observer
   - Strategy
   - Iterator

### Resources

- Design Patterns: Elements of Reusable Object-Oriented Software
- Refactoring.guru pattern catalog

## Level: Mid-Level

### Learning Objectives

- Apply patterns appropriately
- Avoid anti-patterns
- Design for flexibility

### Required Skills

1. **Advanced Patterns**

   - Repository
   - Dependency Injection
   - Middleware

2. **Pattern Selection**

   - Trade-offs
   - When not to use
   - Pattern combinations

3. **Anti-Patterns**

   - God object
   - Spaghetti code
   - Golden hammer

### Optional Skills

- Domain-specific patterns
- Language-specific patterns
- Pattern languages
